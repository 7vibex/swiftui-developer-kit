# Accessibility Claims Risk

Accessibility Nutrition Label claims require repeatable common-task evidence, not isolated labels.

For each claimed feature:

- Name the common task and device family.
- Record the assistive technology and settings used.
- State pass, fail, or not inspected.
- List blocking bugs and unsupported flows.
- Separate Simulator evidence from real-device evidence.

Use `NO-GO` for a claimed feature when a common primary task is blocked. Do not overclaim from static source inspection alone.
