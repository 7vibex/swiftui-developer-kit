# Store Label Readiness

Use this reference when an accessibility audit may inform App Store Accessibility Nutrition Labels or release readiness.

## Common Task Matrix

Before claiming support for a feature, identify common tasks for each supported device family:

- First launch and onboarding.
- Login, account, purchase, and settings when applicable.
- Core app tasks users download the app for.
- Create, edit, save, delete, export, or share user data when these are core tasks.
- Canvas, notebook, planner, tutor, flashcard, and document workflows in StudyOS-style apps.

For each task, evaluate relevant features:

- VoiceOver.
- Voice Control where interaction is complex.
- Larger Text and Dynamic Type.
- Sufficient Contrast and Dark Interface.
- Differentiate Without Color Alone.
- Reduce Motion.
- Captions or Audio Descriptions when media is present.

## Claim Standards

- Pass only when users can perceive, operate, and understand every common task for that feature on the device family being claimed.
- Do not claim support for a feature when a known bug blocks a common task.
- Mark unsupported, partially supported, or untested features clearly.
- Keep evidence current when UI, navigation, controls, or content workflows change.

## Evidence To Capture Without Screenshots

- Code paths for labels, values, traits, and custom actions.
- Dynamic Type layout behavior.
- Reduce Motion and Reduce Transparency behavior.
- Focus order and keyboard/VoiceOver navigation notes.
- Manual test matrix by device, task, and feature.
- Release note or App Store metadata impact.
