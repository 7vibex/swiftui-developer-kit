## Canvas Workflow

`canvas-engine-auditor` gives Codex a focused checklist for iPad canvas apps and document editors. It covers PencilKit bridges, ink tool state, gesture arbitration, screen-to-document coordinate conversion, zoom and offset math, highlighter behavior, layer persistence, PDF annotation flows, undo/redo, performance risks, and regression-test planning.

The planning scope includes:

- `examples/canvas-engine-auditor-example.md`
- `.agents/skills/canvas-engine-auditor/scripts/detect-canvas-risks.sh`
- Canvas, PencilKit, gesture, performance, persistence, and test-matrix references.

## Diagnostics Workflow

`swiftui-diagnostics-builder` helps design privacy-safe diagnostics that are useful to humans and AI agents. It focuses on local logs, breadcrumbs, app-state snapshots, issue-report JSON, MetricKit, signposts, TestFlight feedback, redaction rules, and user-controlled export flows.

The planning scope includes:

- `examples/swiftui-diagnostics-example.md`
- Diagnostics checklists and output contracts.
- Privacy and redaction guidance for report issue workflows.

## Validation Changes

The required verification command remains:

```bash
./scripts/validate-skills.sh
```

Expected result:

```text
Validated the current required skill count, shell scripts, docs, examples, and applicable package checks.
```

Validation will check:

- The current required skill registry is complete; this historical planning note does not fix the count.
- Every `SKILL.md` has YAML frontmatter containing only `name` and `description`.
- Specialist skills include references and an output contract.
- README and `docs/skill-index.md` mention every skill.
- Shell scripts use the expected bash shebang, `set -euo pipefail`, and pass `bash -n`.
- Skill and script folders do not contain blocked destructive or side-effectful commands.
- Screenshot-capable workflows include explicit approval flags and capture-safety language.
- Required examples, command docs, Claude usage docs, demo roadmap, detector roadmap, SECURITY guidance, Apple links, and OpenAI links are present.
- Unfinished-content markers are absent from repo guidance, docs, skills, examples, and scripts.
- `tests/run-tests.sh` passes.

## Upgrade Notes

- Re-run `./scripts/install-local.sh` after pulling the release so the new skill symlinks are present in `~/.agents/skills`.
- Restart Codex only if refreshed skills do not appear in the current session.
- Use `scripts/swiftui-kit.sh list` to confirm the current skill inventory is visible.
- Use `./scripts/validate-skills.sh` before publishing changes to the skill pack. `scripts/swiftui-kit.sh validate` is the equivalent repo-local wrapper.

## Known Limits

- Examples remain fictional and StudyOS-style.
- Screenshot, Appshots, Simulator capture, and Computer Use workflows still require explicit user consent before capture.
- Static scanners are intentionally low-noise and should be treated as triage signals, not complete analysis.
- Diagnostics guidance is privacy-first and local-export focused; remote upload requires separate product, security, and App Store privacy review.
