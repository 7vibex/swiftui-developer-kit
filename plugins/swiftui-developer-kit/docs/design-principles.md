# Apple App Interface Principles

## Content First, Controls Second

Primary app content should stay readable and stable. Controls should support the task without competing with reading, writing, media, maps, documents, forms, or long explanations.

## Apple-Style Restraint

Use platform controls, SF Symbols, predictable navigation, and simple hierarchy. Avoid decorative effects that make dense content harder to scan.

## Hierarchy

Make the main task obvious. Use spacing, typography, navigation structure, and grouping before adding visual effects.

## Accessibility

Review:

- Dynamic Type
- VoiceOver labels and hints
- Contrast
- Tap target size
- Reduce Motion
- Reduce Transparency
- Focus order
- Color-only states

## Liquid Glass Guidance

Use Liquid Glass mostly for chrome and control surfaces:

- Toolbars
- Floating palettes
- Sidebars
- Inspectors
- Contextual panels
- Search bars
- Sheet headers

Avoid Liquid Glass over primary content:

- Document and reading surfaces
- Writing and editing areas
- Media or map content
- Long text explanations
- Forms with many fields
- Code blocks
- Tables
- Dense lists
- Warning, error, or destructive action areas
